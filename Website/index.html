开源 Windows 端点安全工程

把数据保护、威胁检测和策略运营放进同一个终端闭环。

DataProtector 面向企业终端、内网数据和安全团队，将透明文件保护、EDR 遥测、DLP 审计、用户态扫描、沙箱分析、中央策略和安全 U 盘工作流整合在一个可开源验证的 Windows 平台里。

业务价值

不是后台皮肤，而是端点安全产品的业务闭环。

项目把终端文件保护、威胁感知、策略下发、审计取证和移动介质治理放在同一条链路里，适合安全产品原型、企业数据保护研究和 Windows EDR 工程学习。

Sensitive Data

让可信业务继续工作，让敏感数据不轻易离开终端。

透明保护模型按进程、目录和扩展名控制访问视图。研发文档、合同、图纸、客户资料和财务文件可以对授权应用保持可用，同时降低普通工具直接读取、复制、缓存和转存风险。

Threat Visibility

把攻击行为从单点告警变成可追踪故事线。

进程、凭据、注入、横向移动、WebShell、文件和网络传感器统一进入威胁评分与攻击流程。

Policy Operations

让规则、设备、任务、样本和审计统一运营。

中央服务维护策略版本，Agent 心跳同步并回传审计、网络连接、沙箱样本和远程任务结果。

Offline Delivery

让 U 盘成为受控工作区，而不是盲区。

设备授权、介质清点、公开工具区、私有数据区和初始化任务共同组成移动介质治理流程。

核心能力

从文件、进程、网络、样本和介质五个面协同防护。

Transparent Protection

透明文件保护

可信进程规则、受保护扩展名、排除目录、保护标记和影子明文视图，让数据保护不必牺牲业务应用可用性。

EDR Sensors

终端威胁遥测

覆盖 LSASS 访问、注册表 hive、原始磁盘、远程线程、异常镜像加载、ETW 篡改、SMB 投递和 IPC 控制。

Threat Engine

威胁关联引擎

按进程谱系进行评分、衰减、战术映射和故事线记录，支持审计、告警、阻断、网络隔离和终止。

User-mode Scan

用户态静态扫描

内核只捕获可执行文件事件；YARA、哈希信誉、PE 启发式和未来检测引擎在用户态完成判定并回传 verdict。

Sandbox

沙箱与静态分析

Agent 发现可疑 EXE 后上传样本，Server 侧隔离环境运行分析，并结合 Ghidra 与规则评分补充样本上下文。

DLP and USB

DLP 与安全 U 盘

安全目录读取审计、剪贴板与截图控制、可移动介质暂存可见性、USB Crypt 包管理和介质初始化任务构成数据外发治理。

工作闭环

事件在终端发生，判断在用户态演进，策略在中央运营。

这个结构让内核保持轻量，检测内容可以持续更新，运营侧也能拿到可解释的审计与攻击流程。

01

内核捕获事件

minifilter、WFP 和进程回调捕获文件、进程、网络、凭据、横向移动和 WebShell 相关信号。

02

用户态完成重检测

静态扫描、YARA、沙箱、Ghidra 分析和规则评分都留在用户态，避免把复杂检测内容塞进内核。

03

ThreatEngine 关联响应

传感器信号进入进程谱系风险评分，形成攻击故事线，并根据策略执行告警、阻断、隔离或终止。

04

中央策略持续下发

Agent 拉取最新策略并回传审计、网络洞察、样本、设备状态和任务结果，形成运营闭环。

平台组成

单机能验证，中央能运营，前端能展示。

Endpoint

驱动、PolicyApi、本地桥接和 Agent

终端负责透明保护、传感器采集、策略应用、扫描 verdict 执行、DLP 服务、设备清点和心跳同步。

Central

中央 HTTP 服务与策略仓库

服务端维护设备、策略版本、审计、攻击流程、网络洞察、沙箱样本、静态分析记录、USB 包和任务队列。

Operations

WebAdmin、WPF Admin 和 Agent Client

WebAdmin 服务集中运营，WPF Admin 服务本机策略，Agent Client 服务终端用户可见性。

开源定位

适合研究、教学、原型验证和二次开发。

适合谁

Windows 内核开发者、安全产品工程师、企业安全团队、EDR 研究者、DLP 与透明加密方案评估者。

当前价值

仓库覆盖驱动、原生 ABI、中央服务、Agent、Web 控制台、桌面客户端、沙箱、静态分析和 USB 工具。

生产前需要补齐

安装升级、驱动签名、密钥管理、认证加密、规则包发布、兼容性测试、防篡改、权限模型和运维文档。
